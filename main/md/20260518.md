オンライン検出ではどちらが正しい情報化各ノードは知らないため、オフライン検出で用いたロジックをそのまま用いることは難しい。攻撃者は受け取ったすべての経路履歴を改変することがわかっている。複数の経路履歴を時系列で受信していく中で、記録された送信情報が一致する部分と一致しない部分の不整合をうまく絞り込めないか？を考える。

イメージは以下

* チェーン1：S → A → B → C → X → E
    - 経路履歴：{{S,1},{A,1},{B,1},{C,1},{X,1}}

* チェーン2：S → A → B → D → F → E
    - 経路履歴：{{S,0},{A,0},{B,0},{D,0},{F,0}}

チェーン1とチェーン2が情報の伝搬経路としてあるとき、ノードEはチェーン1と2の両方を参照できる状態にある。もちろん前提として、ノードE視点では0と1のどちらが正しい情報かわからないという状況である。そして、チェーン1のノードXが攻撃者であるので、X以前のS → A → B → C → Xの流れの経路履歴中の受信情報のidはすべて0から1に改変されている。ここで、EはノードEはチェーン1と2の両方を参照できる状態にあるが、チェーン1とチェーン2では、S → A → Bのところが共通しているのに、情報idが異なることがわかる。これにより、SとAとBが攻撃者であれば、情報idが一致しているはずであるので、この3ノードは攻撃者ではなく、ノードC,X,D,Fの4ノードが攻撃者であると絞ることができる。ここからまた、別ノードと通信を繰り返すことで攻撃者を絞り込んでいき、閾値より疑惑スコアが上回ったノードを攻撃者として扱う。といったようなオンライン検出の実装は可能か？
