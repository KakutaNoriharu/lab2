# 2026-05-12

## 概要
シミュレーション出力の通信記録（r.csv）を読み込み、情報の伝播経路を可視化した。
偽装情報がどのノードを経由して広がるかを図で確認できるようにした。

## データ
- r.csv: 2,234 件の通信レコード
- 列: No, Time, Source, Target, Content (0=TRUE情報, 1=偽装情報)
- Content=0: 2,208 件 / Content=1: 26 件

## 分析内容

### ノード別通信集計
各ノードについて以下を集計した。
- Content=0 / Content=1 の送信回数（content_0, content_1）
- from（最初に情報を受け取った送信元）/ to（転送先ノード群）の関係
- 伝播ツリーを深さ優先探索で構築し、各ノードのY座標（伝播順）を決定

### 可視化1: 伝播経路図
- X 軸: 受信時刻、Y 軸: 伝播インデックス（初登場順）
- シアン点/線: Content=0 のみ送信したノード（正規）
- 赤点/線: Content=1 のみ送信したノード（攻撃者候補）
- 黒線: Content=0 と Content=1 の両方を送信したノード（正規ノードIDが偽コピー記録に混入したケース）
→ 攻撃者の偽装情報が広がる経路を目視で確認可能

### 可視化2: タイムライン図
- X 軸: 時刻 (Time)、Y 軸: ノードインデックス（初登場順）
- 斜め線で各通信の「送信元→送信先」と発生時刻の対応を表現
- 薄い線 (Content=0) と濃い線 (Content=1) で情報種別を区別
- 偽装情報の発生タイミングと拡散の速さを確認できる
