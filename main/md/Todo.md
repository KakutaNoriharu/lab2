## 長時間・高密度環境における経路履歴の肥大化への対策
提案手法では各ノードが受信した情報に付随する経路履歴（転送ノードのIDと情報内容の組）を保持・比較することで真の情報を選択する．しかし，シミュレーション期間が長くなるほど，また TA 内のノード密度が高くなるほど，情報が多数のノードを経由し経路履歴の長さが際限なく増加する問題が生じる．経路履歴が肥大化すると，各ノードの記憶領域の消費が増大し，情報の送受信時の通信コストも上昇するため，実用的なシステムとしての動作が困難となる可能性がある．\
この問題への対策として，経路履歴に有効期限（TTL: Time To Live）を設け，一定ホップ数または一定時間を超えた古い履歴エントリを自動的に削除する手法が考えられる．あるいは，経路履歴の全エントリを保持するのではなく，経由ノード数のカウントのみを保持する軽量化手法も有効な選択肢である．ただし，単純なカウントのみでは攻撃者が虚偽のカウント値を付与して偽情報を優勢化させる恐れがあるため，カウントの改ざん耐性を確保する仕組みも合わせて検討する必要がある．今後の研究では，履歴の圧縮・削減がもたらす真情報保持率への影響をシミュレーションにより定量評価し，許容できる性能低下の範囲内で最も効率的な履歴管理方式を明らかにする．

## 経路履歴情報を用いた攻撃ノードの特定
現在の提案手法は，経路履歴長の比較によって偽情報を「排除」することを目的としており，偽情報を発信している攻撃ノードそのものを「特定・隔離」する機能は持っていない．しかし，攻撃ノードを特定できれば，そのノードからの情報を以降一切受け付けないブラックリスト方式により，よりロバストな防御が実現できる．\
経路履歴には各転送ノードのIDが記録されているため，複数のノードが受信した偽情報の経路履歴を照合することで，偽情報の発生源となっているノードを特定できる可能性がある．具体的には，異なる経路を経由して届いた複数の偽情報の履歴を比較し，すべての偽情報の経路に共通して出現するノードIDを攻撃ノードの候補とみなす手法が考えられる．ただし，攻撃ノードが自身の ID を偽装したり，複数の ID を使い分けるなりすまし攻撃（Sybil 攻撃）を行う場合には検出が困難となる．そのため，ノード ID の真正性を保証する軽量な認証方式との組み合わせも今後の検討課題として挙げられる．これらの手法をシミュレーション上に実装し，攻撃ノード特定の精度（検出率・誤検出率）および特定までの所要時間を評価することで，実用化に向けた知見を得ることを目指す．
