# 2025-07-16
RAを利用した攻撃について
RAを偽装された情報を受信した場合，ノードは受信時点ではその情報の真偽が確かめられないため，そのRAを避ける方向で行動すると考えることが自然である．
しかし，この場合，攻撃者による偽装されたRAを多くのノードが避け続けることになるため，渋滞などの攻撃による影響を継続的にうけてしまう．
この対策となる要素について実装している．
偽装されたRAについて，その領域を通過したノードがいて，そのノードが偽装されたRAについての情報を受信した場合に，自身が通過できた事実から，嘘の情報であると気づくことができる．
このノードが，偽情報に対して，コメントを付与した状態で配信を行う．
コメントは，コメントしたノードid，コメントした時刻，RAを偽情報と伝える内容の3つの要素からなる．
コメントはIFの蓄積と同様に拡散される．
コメント付きで情報を受信したノードは，コメントを信頼して，RAを偽情報と見なして経路選択を行う．

攻撃者の視点に立つと，RAを無くすことを目的としている場合は，正しいRAに対して噓のコメントを加えることで，RAをなくすことと同様の影響を与えられる．しかし，この場合でも，RAを確認したノードがコメントとして，攻撃者のコメントを否定することで，攻撃による影響を一時的かつ小規模に抑えられると考える．

コメントについて実装する前に，RAについて大きさを変えた場合の影響を確認した．

RAを確認する動作について，実際にノードがRA内にいる場合，そこがRAであると認識できないと考えられる．これは，目立った進入制限，禁止を示すものが置かれていないと考えられるためである．
つまり，ノードはRAを情報を受信する以外の形では検出できない．同時に，攻撃者によってRAが偽装された場合，RAの正しさを検証できない．
